MCP / plugins 21 May 2026
MCP locks its release candidate — stateless core, MCP Apps, Tasks, hardened auth
The Model Context Protocol — the protocol every Cowork plugin and connector speaks — locked its next-spec release candidate. Headline change: stateless at the protocol layer, so remote MCP servers can run behind ordinary load balancers without sticky sessions. Also: MCP Apps (server-rendered UI), Tasks (long-running work), OAuth-aligned auth, formal deprecation policy. Final spec ships 28 July. For Cowork users the effect lands gradually: plugins will get more reliable as SDKs validate.

YouTube May 2026
5-minute setup: LinkedIn lead automation via Cowork + ConnectSafely MCP
Source: YouTube video — “Claude Cowork + LinkedIn = 10,000 Leads/Mo (100% automated)”. Channel and creator unidentified from search snippets; framing and stack details corroborated from ConnectSafely's own integration documentation.
Problem: B2B outreach at scale needs LinkedIn enrichment + personalised messaging + CRM hygiene — too many copy-pastes for a one-person sales motion.
Stack: Cowork (Pro+, persistent memory), ConnectSafely MCP server (remote plugin, LinkedIn API), Cowork's built-in browser automation, file-based ICP definition.
Steps:
• Install Cowork · add ConnectSafely MCP as remote plugin · authenticate LinkedIn through ConnectSafely (5–7 min total).
• Define ICP (ideal customer profile), tone, value prop in persistent memory or a folder.
• Cowork mines LinkedIn for decision-maker contacts across 15+ target companies → structured spreadsheet.
• Per-prospect deep research via web search + LinkedIn activity → appended to row.
• Generate 5-message personalised drip · send connection requests from the same interface.
Ergebnis: ConnectSafely claims 2–3× higher response rate vs. generic outreach. Caveat: this is vendor-adjacent marketing — treat the multiplier as directional, not benchmarked. The 5-minute setup claim is consistent across sources.
Für wen: B2B sales motions where named-account outreach is the bottleneck. For pet-food brands selling into pet stores, veterinary practices and distributors, the ICP-as-file pattern is directly applicable.

Substack May 2026 · ongoing
Simon Willison reverse-engineers the Cowork sandbox — and explains why Skills may matter more than MCP
Creator: Simon Willison (@simonw) · simonwillison.net · simonw.substack.com
Finding 1 — the sandbox: Willison used Claude Code to introspect Cowork's runtime and confirmed it runs each session in an Apple VZVirtualMachine (Apple Virtualization Framework), booting a custom Ubuntu root filesystem per session. Two practical implications: (a) bash runs in an isolated VM, not on your host — safer for letting Claude execute scripts; (b) the VM path differs from the file-tool path for the same files, so absolute paths in bash and the Read/Write tools won't always match. Keep that in mind when chaining bash with file edits.
Finding 2 — skills as the bigger primitive: in a separate post, Willison argues that Claude Skills (folders bundling instructions, scripts and resources that Claude loads when relevant) may be a bigger deal than MCP — because skills are operational inside Cowork, not just useful in chat. In Cowork they shape autonomous work; in chat they're advisory.
Für wen: Anyone wiring scripts into Cowork sessions, or trying to decide whether to build a Skill vs. a MCP server for a new capability. Mostly: build the skill first, add an MCP server only when you need network/external-system access.

Industry blog 11 May 2026
Audie Chamberlain built a real-estate “chief of staff” in Cowork — six tools wired together
Creator: Audie Chamberlain · published Inman (real-estate industry trade) · 11 May 2026. Evergreen (just outside this week's window; included as the strongest concrete enterprise-style build pattern available right now).
Problem: A working real-estate agent's day is fragmented across email, calendar, team chat, CRM, listings, and the web. Switching cost was the bottleneck, not any single tool.
Stack: Cowork connected to Gmail, Google Calendar, Slack, ClickUp, Rechat (real-estate CRM) and Google Chrome — turned into a daily “chief of staff” that outlines priorities, drafts replies, and follows up on stalled threads.
Pattern, generalised: the build is industry-specific (Rechat), but the structure is portable: 5–6 connectors + a /schedule-driven morning brief + Cowork as the orchestrator that synthesises across them. For any operator running a fragmented workday, this is the shape of the build.
Für wen: Owner-operators running businesses with multiple tools and no human chief of staff. The pet-food parallel is direct: substitute Rechat with your supplier portals / pricing data, Slack stays, Gmail stays, ClickUp becomes whatever you use for ops.

Concrete, replicable workflows — if you have one hour this weekend.
Try this ~30 minutes
Extend your /schedule lineup with morning email triage
Why this one: You already run eight scheduled briefings. The lowest-friction addition with daily compounding value is morning email triage — not a summary, a triage (urgent / waiting on you / informational / archive).
Pattern (cribbed from the AIBlewMyMind post above): /schedule a 7:30 task · pull last 24h of Gmail via the Gmail connector · classify each thread into four buckets with rationale · surface only urgent + waiting-on-you with proposed reply drafts (no auto-send) · HTML to News folder, same pattern as your briefings.
Setup honesty: first run will misclassify — budget 15 minutes after run #1 to write rules into the prompt (e.g. “always treat invoices from supplier X as urgent”). Quality stabilises by run #3.
Try this ~1 hour
Build a private pet-food plugin from the knowledge-work-plugins template
Why this one: You run two pet-food companies (Alphapet, SmartPetPro) with recurring cost-of-goods tracking, supplier monitoring and competitive checks. Bundle them into a private plugin and you get one install, shared skills across both businesses, slash-commands like /petfood-week, and a clean handoff if anyone else ever joins.
Steps: clone Anthropic's open-source knowledge-work-plugins repo · strip what you don't need, keep the scaffolding · add two skills (cost-of-goods-watch and supplier-pulse) · wire a /petfood-week slash command running both · test in a sandbox folder before adding to your main Cowork install.
Honest caveat: first plugin build is a 2–3 hour learning curve, not 1 hour. Budget accordingly. Reusability return is real.

Why this one: The real-estate build above is the cleanest example of multi-connector orchestration in production. Connectors swap; structure doesn't.
Map for pet-food: Gmail/Outlook stays · Google Calendar stays · Slack/Teams stays · ClickUp/Asana/Notion (whichever you use for ops) · swap Rechat (real-estate CRM) for your supplier portals or B2B CRM (HubSpot if relevant) · Chrome stays for browser-based supplier portals without APIs.
First milestone: a daily 7:45 brief that synthesises overnight email + today's calendar + open ops tasks into one page. Don't build the full chief-of-staff in one sitting — start with the morning brief, add layers weekly.

💡 Editor's view
1. If you use Cowork only as a chat interface, you use roughly 20% of the tool. The lever that turns Cowork from “helpful sometimes” into “running things while you sleep” is the combination of /schedule + a small plugin bundle + persistent memory pointed at the right folders. Every praxis story this week converges on that pattern. If you have a /schedule list shorter than five tasks, the highest-ROI hour you'll spend on AI tooling this weekend is “try this week” #1.
2. Vertical bundles are the strategy, not skills. Finance (5 May), legal (12 May), small business (13 May), marketing ops (18 May). The unit Anthropic ships is no longer “a skill” or “a connector” — it's a pre-composed bundle for a job function. If you're building anything multi-user, mirror that shape: bundle, not loose instructions.
3. Plugin reliability gets noticeably better through Q3. The MCP stateless-core release candidate is the biggest infrastructure change since the protocol launched. Combined with Stainless in-house, the friction of building and shipping a plugin should drop meaningfully. If you've been putting off a private build because the plumbing felt fragile, mid-summer is when that excuse expires.
Quiet pattern to watch: how fast community marketplaces (a16z, claudemarketplaces.com) grow versus Anthropic's official directory. If community outpaces official, Cowork becomes a platform other people build businesses on top of — not just an Anthropic product. That's the agentic-OS moment.

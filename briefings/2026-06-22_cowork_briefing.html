MCPSpecRC · final 28 Jul

The biggest MCP revision since launch hits Release Candidate

The MCP 2026-07-28 spec is in release candidate — its largest revision yet. The headline is a stateless core that runs on ordinary HTTP (a remote server that needed sticky sessions and a shared store can now sit behind a plain round-robin load balancer), plus MCP Apps (server-rendered UIs), a Tasks extension for long-running work, and OAuth/OIDC-aligned auth. For Cowork users this is upstream plumbing, but it's why connectors keep getting easier to stand up and run reliably.

"Claude can now run scheduled tasks" — Khe Hy's 2-minute setup pattern

Khe Hy · khemaridh.substack.com (RadReads)

Problem: Recurring knowledge-work (digests, cleanups) that you keep doing by hand.

Stack: Cowork Desktop · the new /schedule command · no code, no API keys.

  • Write the task as a normal prompt and run it once manually.
  • Check the output is what you want.
  • Type /schedule; Claude walks you through cadence (hourly / daily / weekly / weekdays / on-demand) and time.
  • It runs unattended on that cadence and delivers the artifact.

Ergebnis: The whole setup is ~2 minutes. Honest limit: you must validate the manual run first — a vague prompt schedules a vague output forever.

Für wen: Anyone with a weekly cadence already in their head (exactly your briefing workflow).

Simon Willison reverse-engineers the Cowork sandbox (and the prompt-injection risk)

Simon Willison · simonwillison.net

Problem: What is Cowork actually running on, and how exposed is it?

Stack: Claude Code → reverse-engineered the Claude macOS Electron app; had Cowork inspect its own environment.

A scheduled "retail shelf-price watch" for your own SKUs and 2–3 competitors

You already run a weekly commodity-cost briefing; the missing mirror is the sell-side. Stand up a scheduled task that checks DACH retail/marketplace listing prices for your hero SKUs and a couple of competitors, and writes a one-page weekly delta to your News folder — the same pattern as your cost briefings, pointed downstream.

  • Draft the prompt: the SKU list, the retailers/marketplaces to check, and "flag any price move > 3% WoW."
  • Wire a web/retail MCP connector (or have it read a price-export you drop in a mounted folder, to stay inside terms of service).
  • Run once manually, verify the table, then /schedule it Monday 09:30 — right after the cost briefing, so cost and shelf price sit side by side.

Editor's view

The lever this week is scheduling, and the unlock underneath it is managed connectors

1. Scheduled tasks are the thing that converts Cowork from "helpful sometimes" to "running things while you sleep." Every strong praxis source this week — Khe Hy, Lenny's engineer interview, the Substack recipe posts — lands on the same move: validate a prompt once, /schedule it, walk away. You're already ahead of the curve here; the new material is mostly confirmation that your operating model is the recommended one.

2. The quiet enabler is enterprise-managed connectors. Zero-touch MCP provisioning is what makes scheduled automations safe and repeatable across more than one person — it's the difference between a personal hack and a team capability. If Alphapet/SmartPetPro ever puts a second person on Cowork, this is the feature that matters.

3. Heed the one honest caveat in the praxis pile. Willison's prompt-injection warning is the adult-in-the-room note: the more you automate against untrusted inputs (inboxes, web pages), the more you must scope connector permissions and keep send/write actions behind manual approval — exactly how the "Try this week" builds above are structured.

Wildcard to track: the MCP 2026-07-28 stateless-core spec lands 28 July. When remote MCP servers get trivial to run reliably, the connector ecosystem behind Cowork gets a step-change — watch whether that pulls more vertical MCPs (retail pricing, vet/distributor data) into reach for a business like yours.

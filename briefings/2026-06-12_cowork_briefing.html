Briefing summary
Fable 5 lands in Cowork — your scheduled tasks just got a Mythos-class brain, free until June 22
The week belonged to one event: Anthropic shipped Claude Fable 5 and Mythos 5 on June 9, and Fable 5 is live inside Cowork at no extra cost on paid plans until June 22. The praxis ecosystem pivoted accordingly — the best hands-on content this week is model-evaluation discipline (Simon Willison's 5.5-hour test, Paweł Huryn's 1,000 timed runs) rather than new workflow builds, so the tutorial shelf is thinner than usual but the blogs section is strong. Quieter but durable: Cowork task management arrived on iOS/Android, and the MCP (Model Context Protocol) spec went stateless in a release candidate — the plumbing behind every Cowork connector just got more reliable.
 
●  What shipped this week
ANTHROPIC Jun 9
Claude Fable 5 & Mythos 5: first Mythos-class models — and Fable 5 runs inside Cowork today
Anthropic launched a new model tier above Opus. Fable 5 is the publicly available variant: always-on adaptive thinking, a 1-million-token context window, 128K output tokens, and state-of-the-art results on nearly every tested benchmark. Safety architecture is novel — classifiers screen high-risk queries and reroute roughly 5% of sessions to Opus 4.8 with a notification. Mythos 5 shares the capabilities without those classifiers and stays restricted.
The Cowork-relevant part: Fable 5 is selectable across Claude.ai, Claude Code and Cowork — including as the model on a scheduled task. Secondary reporting says it is included on subscription plans until June 22, then billed extra (see banner below).

SUBSTRATE Jun 10–11
Claude Code 2.1.172/173: nested sub-agents five levels deep, plus plugin search
This is the CLI (command-line) substrate, not Cowork itself — but the pattern matters: sub-agents can now spawn their own sub-agents up to 5 levels deep, which is how complex delegation chains (research → verify → synthesize) get built before they surface in Cowork. Also new: a search bar when browsing marketplace plugins in /plugin, Fable 5 "[1m]" model-name normalization (1M context is now default), and smarter AWS (Amazon Web Services) region handling for Bedrock users.

COWORK Early Jun (rollout)
Cowork tasks now manageable from your phone: persistent agent thread on iOS/Android
Pro and Max subscribers get a persistent agent thread accessible from Claude Desktop or the iOS/Android apps to check on and steer running Cowork tasks. Rollout started with Max plans in early June, Pro over the following days. Caveat unchanged: desktop scheduled tasks still only fire while your machine is awake with the app open — the mobile thread is for managing tasks, not for running them serverless.

MCP / INFRA Jun 2026
MCP goes stateless: release candidate for the July 28 spec is out
The Model Context Protocol (MCP) — the connector standard underneath every Cowork integration — published its release candidate for the 2026-07-28 specification: a stateless protocol core, an Extensions framework, Tasks, MCP Apps, authorization hardening and a formal deprecation policy. Practical translation: remote MCP servers no longer need sticky sessions, so the hosted connectors Cowork relies on (LinkedIn tools, Stripe, CRM systems) get cheaper to run and less likely to drop mid-task. Final spec ships July 28.

●  Editor's view
1. The lever that converts Cowork from helpful-sometimes to running-things-while-you-sleep this week is model selection per scheduled task. Fable 5's profile — slow, thorough, 1M context, adaptive thinking that doesn't waste budget on easy runs — is built for unattended pipelines, not chat. A free ten-day trial window on exactly that workload is rare; use it on the two briefings where dropped threads cost you most.
2. The praxis ecosystem leveled up its methods this week: Willison's launch-day deep test and Huryn's 1,000 timed runs are model-evaluation discipline applied by practitioners, not benchmarketing. The community is no longer asking "what is Cowork" — it's asking "which model earns which task at which price." That is the question of a maturing tool category.
3. Mobile task steering plus the MCP stateless spec are the same story from two ends: the agent loop is detaching from your desk. Phone-side management now; cheaper, more reliable hosted connectors by late July. The remaining gap is serverless scheduled execution — tasks that run with the laptop closed.
Wildcard worth tracking: Claude Code's nested sub-agents (five levels deep) haven't surfaced in Cowork yet — when they do, a single scheduled briefing could fan out into research/verify/synthesize teams. Watch for it in the next two release cycles.

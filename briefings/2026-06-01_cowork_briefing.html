Briefing summary
The week Cowork stopped being a "look what it can do" demo and became a substrate practitioners are building production loops on.
Anthropic shipped Opus 4.8 with dynamic workflows and a 2.5x-speed Fast Mode, raised $65B at a $965B valuation, and locked the MCP 2026-07-28 spec RC. But the more interesting signal is downstream: scheduled tasks crossed the "I automated my actual job" threshold for several creators, ConnectSafely's LinkedIn MCP turned cold outreach into a 5-minute setup, and Anna Mills published the cleanest worked example yet of Cowork building and self-testing its own skill. The lever this week isn't a feature — it's the realization that the praxis pattern of "scheduled task + 1 MCP + 1 skill" is now teachable in under 15 minutes.

SUBSTRATE · MCP May 21 (locked) · pub Jul 28
MCP 2026-07-28 spec release candidate locked — stateless core, Extensions, Tasks, MCP Apps
MCP (Model Context Protocol — the connector layer Cowork plugins ride on) is now stateless at the protocol layer. Streamable HTTP transport requires Mcp-Method and Mcp-Name headers so gateways, load balancers, and rate-limiters can route on the operation without inspecting the body. Extensions framework, Tasks, MCP Apps, authorization hardening, and a formal deprecation policy round out the RC. Ten-week validation window opens; final spec publishes 28 Jul.
Why it matters for Cowork users now: any custom MCP server you build in the next quarter should target the stateless transport — otherwise it's tech debt the moment the spec lands.

SUBSTACK last 14d
Get Cowork to build & self-test its own LinkedIn-search skill
Anna Mills · annamills.substack.com
Problem. Existing LinkedIn-search workflows in Cowork are brittle — they break the moment LinkedIn changes its rendering. Mills wanted a skill she could maintain by talking to Claude, not by reading her own old prompts six weeks later.
Stack. Cowork · Chrome MCP / browser tool · the skill-creator skill · a "test harness" prompt she keeps in a folder she granted Cowork.
Steps.
  1. Ask Cowork to draft a SKILL.md for "find me LinkedIn profiles matching criteria X."
  2. Save it, then in the same chat tell Cowork to use the skill on three known test cases she pre-wrote.
  3. Have Cowork score itself against expected output, then patch the skill.
  4. Repeat until the test cases pass without intervention.
  5. Schedule a weekly run of the test harness so drift is caught early.
Ergebnis. Working skill at iteration 4. Mills is candid that the self-eval loop is only as good as the test cases; one bad test caused Cowork to overfit. The Karpathy-style "evals first" framing is the lesson, not the LinkedIn part.
Für wen. Anyone shipping >1 custom skill. The pattern (skill + test harness + self-eval loop) is the most reusable thing in the praxis literature this month.
C-Score 4/5 substack

SUBSTACK last 30d
The "Karpathy loop" — a no-code self-improving Cowork pattern
Karo Zieminski · karozieminski.substack.com
Problem. Skills go stale. Most teams write a skill once and never edit it. Zieminski wants Cowork to fix its own skills by inspecting its own failed outputs.
Stack. Cowork · a "logs" folder Cowork writes failed-run notes to · a weekly /schedule that opens the logs and rewrites the relevant skill.
Steps.
  1. Add to every skill: "if you encounter an unexpected case, append a one-line note to /logs/[skill-name].md and continue."
  2. Weekly scheduled task: "open /logs, group recurring failure modes, propose 1–3 edits to the skill, write the diff to /proposals."
  3. Human review: accept/reject in 5 min once a week.
  4. Cowork applies accepted edits.
Ergebnis. Zieminski reports compounding skill quality across 6 weeks. Caveat: Cowork sometimes "fixes" a skill toward the most recent failure at the expense of older successful behavior. The /proposals review step is non-optional.
Für wen. Anyone maintaining 3+ custom skills. This is the most reusable meta-pattern of the month — pair it with Anna Mills' eval-harness skill.

Editor's view
Three things worth holding
1. The praxis lever this quarter is skill + test harness + scheduled self-edit, not any single new feature. Mills' eval harness and Zieminski's Karpathy loop are two halves of the same pattern. The teams who internalize "skills evolve weekly under a /schedule that reads /logs" will out-execute teams who write skills once and forget them. This is the lever that converts Cowork from helpful-sometimes to compounding-while-you-sleep.
2. Opus 4.8 Fast Mode is the biggest economics shift inside Cowork in months. Recurring scheduled tasks are exactly the workload Fast Mode is priced for. The right next move is a single A/B run on your AI weekly briefing, not a wholesale switch. If parity, the briefing pipeline switches; if not, you've still learned the failure modes cheaply.
3. "Laptop must be awake" is now the dominant /schedule failure mode — and Cursor just made it a competitive frame. Dispatch Mode helps but doesn't solve it. If Anthropic doesn't ship a cloud-execution option for Cowork in Q3, enterprise procurement teams will start asking the question. Worth tracking.
Wildcard to watch: Anthropic's $47B revenue run rate implies Cowork plus Claude Code are pulling more weight in the mix than published narrative suggests. The next vertical bundle — and the next enterprise marquee win after PwC and KPMG — will tell us which sector Anthropic believes converts the fastest.

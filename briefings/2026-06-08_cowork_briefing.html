A loud week for the company, a quiet one for fresh praxis — and the most useful lever is still hiding in a Cowork support article

Honest read on praxis: very little net-new walkthrough content carried a 2–8 Jun datestamp. So this edition leans on the strongest evergreen patterns — Karo Zieminski's self-improving scheduled-task loop, Daria Cupareanu's six scheduled-task recipes, and Simon Willison's still-definitive first-impressions teardown. The lever worth your weekend is the self-improving loop: Cowork already rewrites a scheduled task's own prompt after the first run, and you can steer that rewrite toward quality, not just connector plumbing. That is the difference between a briefing that ages and one that compounds.

How I use Claude Skills + Cowork to run my whole LinkedIn content engine
Live demo walkthrough · Maven session
Problem
Content creation is a repeating multi-step process (ideate → draft → match voice → format per channel) that gets re-explained to a chatbot every single time.
Stack
A custom "write like me" Skill (voice fingerprint) · a Cowork Project holding past posts and source material · Cowork doing the multi-step run instead of one-shot prompting.
Steps
1. Build a Skill that encodes your voice from a handful of real samples.
2. Drop reference material into a Project so context persists.
3. Point Cowork at a topic; it drafts, self-checks against the voice Skill, and formats for the channel.
4. Review and ship — the human stays in the loop on final approval.
Ergebnis
The unlock is the Skill-as-voice pattern: once encoded, output stops sounding generic. Caveat — a voice Skill is only as good as the samples; thin or off-tone samples produce confident-but-wrong drafts.

Substack ★ Lead pattern 9 Apr 2026
A self-improving Cowork loop: let a scheduled task rewrite its own prompt toward quality, not just plumbing
Karo Zieminski · @karozieminski · Product with Attitude (Substack)
Problem
Every recurring workflow decays — files move, conventions change, new edge cases pile up — but the prompt stays frozen, and you only fix what you've already seen break.
The hidden mechanic
Cowork already rewrites a scheduled task's prompt after its first run. Because each run is an isolated session with no memory of the conversation that created it, the rewritten prompt is the task's continuity — effectively a CLAUDE.md the task wrote for itself. Natively, that rewrite optimises for connector accuracy (paths resolve, channels map). Zieminski's move: add an improvement directive so the rewrite optimises for quality and relevance instead.
Stack
No code, no terminal. Three pieces: a context.md in the project folder · folder instructions · an improvement directive in the scheduled-task prompt. Karpathy's auto-research loop, adapted to Cowork's visual interface.
Steps
1. Create a project with a context.md that holds the task's evolving playbook.
2. Write the scheduled task so each run ends by reflecting on what could be sharper and updating context.md.
3. Add the directive that points the rewrite at output quality, not just connector mapping.
4. Let it run; after 10+ cycles context.md becomes a playbook Claude wrote for its own workflow.
Ergebnis
By the second run the prompt is already more precise than what you wrote. Honest limits the author flags: instruction drift after 20–30 runs and the need for a guardrail/eval rubric to keep the loop from optimising itself sideways — the full guardrail system sits behind her paywall.
Für wen
Anyone running recurring scheduled briefings — i.e. exactly your weekly AI, pet-food commodity, and Cowork briefings. This is the single highest-leverage pattern in this edition.

Weblog 12 Jan 2026 · foundational
The definitive teardown: what Cowork actually is, and where prompt injection bites
Simon Willison · @simonw · simonwillison.net
Why it still matters
Five months on, this remains the clearest mental model: Cowork is "Claude Code for the rest of your work" — regular Claude Code wrapped in a friendlier interface with a filesystem sandbox configured for you. Willison reverse-engineered the macOS Electron app and confirmed it runs an Ubuntu VM via Apple's Virtualization framework (VZVirtualMachine) booting a custom Linux root filesystem.
The caveat to keep
His central warning ages well: a general agent with real-world capabilities is a live prompt-injection surface. Every connector you add, every web page a scheduled task reads, is potential instruction. The 6 Jun cross-session-messaging hardening in Claude Code is the substrate slowly closing exactly this class of gap.

Turn your pet-food commodity briefing into a self-improving loop
You already run a weekly cost-of-goods commodity sweep. Add Zieminski's mechanic to it: create a context.md in that briefing's project folder, and append one instruction to the scheduled task — "at the end of each run, note in context.md which sources gave usable price/availability reads this week and which were noise, and adjust next run's source list and structure accordingly." After 5–6 weeks the task will have learned, for example, that MATIF rapeseed and TTF gas pages resolve cleanly while a given renderer's page rarely updates — and it will reweight on its own. Add a one-line guardrail: "never drop a named supplier or protein cluster from coverage; only reorder and reprioritise." That protects your three-cluster structure from drift.

A daily B2B outreach prep task for the pet-store / vet channel
Combine the ConnectSafely LinkedIn pattern with a scheduled task. Store an ICP for your two best-fit buyer types (independent premium pet stores; small-animal vet practices) in a Project, plus your tone and a few winning past messages as a voice Skill. This keeps you on ConnectSafely's sanctioned API path, keeps the human on final approval (the right call given Willison's prompt-injection caveat), and removes the daily re-briefing tax.
